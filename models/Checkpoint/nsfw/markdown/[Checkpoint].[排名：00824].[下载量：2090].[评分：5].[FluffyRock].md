### 三、详情
<p>Official-enough Civitai upload of some of the common/popular/newer FluffyRock models. This is mostly being done to allow other posts and models to be able to properly reference the original models. Lodestone Rock trained these models.</p><p>More versions will be added over time. Leave comment if you need a particular older epoch posted. Newer models will be uploaded here as I have the time to upload and make sample images. Original Hugging Face repo will always be most current.</p><p></p><p>Note, the info here is incomplete. This will be improved <em>eventually</em>.</p><p></p><h3 id="heading-9194"><strong>Currently recommended version:</strong> </h3><p>Latest epoch of 3m or terminal-snr.</p><p>Or vpred with a suitable setup to sample it. Results have been pretty impressive, but requires a sampler with cfg rescale support. </p><p></p><h3 id="heading-9195"><strong>Quick breakdown on each model line here.</strong></h3><p><strong>1088-megares</strong>: Trained on highres dataset up to 1088px. Considered finished at e27 as it had plateaued and efforts moved to other methods.</p><p><strong>1088-megares-offset-noise</strong>: Same as above, but with additional epochs with offset noise. Helps increase dynamic lighting range of dark and light parts of images, ie. darker darks can be possible.</p><p><strong>1088-megares-offset-noise-3M</strong>: Same as above with larger &gt;3 million image dataset. Able to understand more concepts.</p><p><strong>1088-megares-terminal-snr</strong>: Similar goal to offset noise, but technically different method (see the FluffyRock thread on the FD Discord thread on FD for discussion, its there somewhere. TODO: actually learn that myself and put that info here :3).</p><p><strong>1088-megares-terminal-snr-vpred:</strong> Uses v parameterization. This is the newest branch and can gives really good results. Still experimental and using it requires additional steps.</p><p>Requires config file and cfg rescale. For A1111 and Vlad fork, can use <a target="_blank" rel="ugc" href="https://github.com/Seshelle/CFG_Rescale_webui">CFG_Rescale_webui extension</a> or pull the cfg rescale PR from A1111 (unless it has since been merged upstream).</p><p><strong>e6laion</strong>: Trained on a dataset of e6 and laion. It is relearning things that base SD1.5 had. Much experimental and does not have many epochs yet. Is not uploaded here yet. Can be download from the <a target="_blank" rel="ugc" href="https://huggingface.co/lodestones/furryrock-model-safetensors/tree/main/e6laion">HuggingFace repo</a>.</p><p></p><h3 id="heading-9196"><strong>VPred troubleshooting:</strong></h3><p>Output just noise/cloud: Missing config file.</p><p>Output too dark: Turn up cfg rescale.</p><p>Some samplers may not work as cfg rescale support is not complete as of yet.</p><p></p><h3 id="heading-9197">Training LoRAs:</h3><p>Previously, e27 was more recommended as the model to train against as the results would be more portable to the other FR model branches at the time. This is outdated.</p><p>LoRA's trained on any recent branch of FR have had decent portablity between branches lately in my personal experience. But training against the model you plan to sample is going to likely have the best results.</p><p>Noise-offset models may require training with noise offset &gt;0 to have good results, though these LoRAs may not work well on other models that do not use noise offset.</p><p>vpred requires training with v_parameterization enabled. kohya_ss will complain about using that on v1, ignore that, nobody expected v parameterization would get used on v1.</p><p>Ask in the Discord for help.</p><p></p><h3 id="heading-9198">Links and resources:</h3><p><a target="_blank" rel="ugc" href="https://civitai.com/models/92527/fluffyrock-pre-3m-autocomplete">Tag Autocomplete File</a></p><p><a target="_blank" rel="ugc" href="https://huggingface.co/lodestones/furryrock-model-safetensors/tree/main/">Hugging Face Repo</a> Contains every version of every model line.
